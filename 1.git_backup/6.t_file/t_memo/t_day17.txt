2) 예외처리
	예외가 발생했을 때 프로그램이 비정상적으로 종료되는 것을 막고 정상적으로 실행될 수 있도록 처리하는 것을 의미

	try{

	}catch(예외클래스명 참조변수){
		예외 발생 시 실행할 코드;
	}catch(예외클래스명 참조변수){
		예외 발생 시 실행할 코드;
	}finally{
		예외발생 유무에 상관없이 무조건 실행할코드(생략가능)
	}

	.printStackTrace()	예외 발생지점을 추적하여 단계별로 모두 출력함

(4) 메소드 보기

	Deprecated Methods : 향후 제거된 메소드 목록

2) 모듈, 패키지, 라이브러리, 프레임워크
- 모듈(Module)
	가장 작은 단위
	
	jdk9부터 도입된 모듈 시스템
	코드를 논리적으로 그룹화하고 모듈간의 의존성을 관리하는 방법을 제공한다
	모듈은 패키지와 라이브러리의 개념을 확장하고 런타임에서 모듈의 논리적인 단위로 코드를 로드하고 관리할 수 있다
	코드의 의존성을 더 명확히 하고 런타임 모듈 시스템을 통해 더 격리된 환경을 제공한다

- 패키지(Package)
	관련된 클래스 및 인터페이스를 그룹화하는 방법을 제공한다
	패키지는 디렉터리 구조와 일치하고 패키지 내의 클래스들을 동일한 패키지 이름 공간안에 있어야한다
	코드를 구조화하고 이름 충돌을 방지하는데 도움을 준다

- 라이브러리(Library)
	jar(java Archive) 압축파일 형태로 존재한다
	jar 파일에는 클래스와 인터페이스의 바이트 코드(.class) 파일들이 압축되어있다

- 프레임워크(Framework)
	어플리케이션의 전체 구조와 실행 흐름을 제공하는 기반 환경
	대표적인 java 프레임워크 => Spring / Spring Boot, Struts, Hibernate(ORM 프레임워크)

=> 클래스/인터페이스 -> 패키지 -> 모듈(JDK 9+) -> 라이브러리(JAR) -> 프레임워크

2. 스트림 API
	Stream API는 JAVA8에서 도입된 기능으로 컬렉션 데이터를 선언적이고 함수형 방식으로 처리하기 위한 API
	여러개의 데이터를 흐름처럼 하나씩 처리하기 위한 도구
	
	- 스트림은 데이터를 저장하지 않는다
	- 스트림은 데이터를 하나씩 흘려보내며 처리한다

	스트림 = 데이터 처리 파이프
	데이터 -> 가공 -> 결과

1) 스트림 API
	선언적 : 무엇(what)을 할 지만 표현
	데이터를 선언적이고 함수형 방식으로 처리한다
	반복문(for문, while문) 없이도 데이터 처리가 가능하다
	데이터의 처리 과정에 집중할 수 있도록 해준다
	"스트림은 데이터를 저장하는 구조가 아니라 데이터를 처리하는 도구이다"
	스트림은 여러개의 데이터를 하나씩 흘려보내며 처리한다
	
2) 스트림 연산구조
	스트림 생성 -> 중간연산 -> 최종연산
		
(1) 스트림 생성(Stream Creation)
	데이터로부터 스트림을 만드는 단계
	이 단계에서는 아직 아무 처리도 일어나지 않는다	

	Stream.of(1, 2, 3)

	int[] ar = {1, 2, 3};
	Arrays.stream(ar)	//배열을 스트림으로 변환

(2) 중간연산(Intermediate Operation)
	스트림을 가공(변환)하는 단계
	결과는 새로운 스트림
	지연실행(Lazy) 된다
	이 단계에서는 아직 실행되지 않는다
	
	- filter()	조건으로 거르기(true / false)
	- map()		값 변환
	- sorted()	정렬
	- distinct()	중복 제거

(3) 최종연산(Terminal Operation)
	스트림을 소비하는 단계
	실제 실행이 발생하는 시점
	결과를 출력하거나 값으로 만든다
	최종연산이 호출되는 순간 스트림 전체가 실행된다
	
	- forEach()	
	- count()
	- reduce()
	- findFirst()

3) 스트림 특징
	데이터를 저장하지 않는다
	데이터를 하나씩 처리한다
	한 번만 사용할 수 있다
	
4) 지연실행 장점
	불필요한 연산 제거
	성능 최적화(필요한 만큼만 계산, 메모리 사용 최소화, 대용량 데이터에 유리)

3. Object 클래스
	
	java.lang	- Object 클래스	모든 클래스의 최상위 부모 클래스
			- System 클래스	키보드로부터 데이터를 입력받을 때 사용
					모니터(콘솔)로 출력하기 위해 사용
			- 문자열관련
			  String	문자열을 저장하고 조작할 때 사용
			  StringBuilder	효율적인 문자열 조작 기능이 필요할 때 사용
			  java.util.StringTokenizer	구분자로 연결된 문자열을 분리할 때 사용
			- 포장관련(Wrapper) 기본 타입의 값을 포장할 때 사용, 문자열을 기본 타입으로 변환 시 사용
				Byte, Short, Integer, Character, Float, Double, Boolean
			- Math		수학 계산이 필요할 때 사용
			- Class		클래스의 메타 정보(이름, 구성멤버) 등 조사할 때 사용

1) Object 클래스의 메소드
	메소드				용도
	boolen equals(Object obj)	객체의 번지를 비교하고 결과를 리턴
	int hashCode()			객체의 해시코드를 리턴
	String toString()		객체 문자 정보 리턴

2) toString() 메소드
	참조값을 문자열 형태로 반환한다
	클래스경로와이름@16진수로 변환된(해시코드)
		toHexString(hashCode())	=> 해시값, 16진수로 변환된(해시코드)값
	
	참조변수를 출력메소드로 출력하면 항상 toString()이 사용된다 => 참조값이 나온 이유
	toString 재정의를 통해 출력되는 문자열의 형태를 바꾸면 된다

3) equals() 메소드
	Object의 equals()는 사실 관계연산자(==)와 같다(주소값 비교)
	String의 equalse()는 재정의되어 값을 비교한다
	객체를 비교하고 싶다면 비교 기준을 재정의해서 사용할 수 있다

4) hashCode() 메소드
	객체 해시코드 : 객체를 식별하는 정수를 의미
	Object의 hashCode() 메소드는 객체의 메모리 번지를 이용해서 해시코드를 생성하기 때문에 객체마다 다른 정수값을 리턴한다
	equals() 메소드와 마찬가지로 hashCode()메소드 역시 객체의 데이터를 기준으로 재정의해서 새로운 정수값을 리턴하도록 하는것이
	일반적인 방식이다
	객체가 다르다 할지라도 내부 데이터가 동일하다면 같은 정수값을 리턴하기 위함이다
	자바는 두 객체가 동등함을 비교할 때 hashCode()와 equals()메소드를 같이 사용한다

1) 생성 패턴(Creational Pattern)
	객체 생성 방식에 관한 패턴

- 싱글톤(Singleton) ★★★
	객체를 하나만 생성하도록 보장하는 패턴
	생성자 private, 객체 하나만 공유, 전역 접근 가능
	단일객체, 전역 인스턴스, 메모리 절약

- 팩토리 메소드(Factory Method) ★★
	객체 생성을 서브 클래스에 위임하는 패턴

	객체 생성 책임 분리, 다형성을 활용
	객체생성 캡슐화, 생성 책임 분리

- 추상 팩토리(Abstract Factory) ★★
	관련 있는 객체들을 묶어서 생성하는 패턴

	여러 객체를 하나의 그룹으로 생성, 제품군 생성
	객체 집합 생성, 제품군

- 빌더(Builder) ★★★
	복잡한 객체를 단계적으로 생성하는 패턴

	생성과정과 표현 분리, 불변 객체 생성 가능
	setter 대체, 단계적 생성

- 프로토타입(Prototype) ★
	기존 객체를 복사해서 새로운 객체를 생성하는 패턴
	객체 복제, clone()

2) 구조패턴(Structural Pattern)
	클래스, 객체 구조에 관한 패턴

- Adapter(어댑터, 적응자 패턴) ★★★
	호환되지 않는 인터페이스를 연결하는 패턴
	중간에서 맞춰주는 역할
	변환기, 인터페이스 호환

- Bridge(브릿지 패턴)
	구현부과 추상층을 분리
	기능과 구현 분리, 구현부에서 추상 계층을 분리, 독립적 확장

- Composite(컴포지트 패턴)
	객체를 트리구조로 구성하여 부분-전체를 동일하게 처리
	트리구조, 부분-전체, 단일객체와 복합객체 모두 동일하게 다루도록 함

- Decorator(데코레이터 패턴) ★★★
	기존 객체에 기능을 동적으로 추가하는 패턴
	주어진 상황 및 용도에 따라 객체에 책임을 덧붙임
	기능확장, 상속 대체

- Facade(퍼사드) ★★★
	복잡한 시스템을 단순한 인터페이스로 제공하는 패턴
	결합도를 낮춰 시스템 구조에 대한 파악을 쉽게함
	창구역할, 복잡성 은닉

- Flyweight(플라이웨이트)
	공유를 통해 메모리를 절약하는 패턴
	객체 공유, 메모리 절약

- Proxy(프록시)
	대리 객체를 통해 접근을 제어하는 패턴
	대리자, 접근 제어

3) 행위패턴(Behavioral Pattern)
	객체 간 상호 작용과 책임 분배

- Template Method(템플릿 메소드) ★★★
	정처기 진짜 찐 단골
	알고리즘의 구조를 정의하고 일부 구현을 서브 클래스에 맡기는 패턴
	알고리즘 골격, 상속 기반

- Strategy(전략) ★★★
	알고리즘을 캡슐화하여 교체 가능하게 만드는 패턴
	행위변경, if제거

- Observer(옵저버) ★★★
	객체 상태 변화시 자동으로 통지하는 패턴
	한 객체의 상태가 바뀌면 그 객체에 의존하는 다른 객체들한테 연락이 가고 자동으로 내용이 갱신되는 방법으로 일대다 의존성을 가지는
	디자인패턴과 서로 상호작용을 하는 객체 사이에서는 가능하면 느슨하게 결합하는 디자인 패턴
	이벤트 처리, 1:N 관계

- Command(커맨드)
	요청을 객체로 캡슐화하는 패턴
	요청 객체화, 실행 취소 가능

- State(상태)
	객체의 상태에 따라 행동을 변경하는 패턴
	상태 객체, 조건문 제거

- Iterator(이터레이터) 
	컬렉션 내부 구조를 노출하지 않고 순회하는 패턴
	순차 접근, 내부 구조 은닉

- Mediator(중재자)
	객체 간 복잡한 통신을 중재 객체로 집중

- Memento(메멘토)
	객체의 상태를 저장하고 복원하는 패턴
	undo 기능을 개발할 때 사용하는 패턴

- Chain of Responsibility(책임 연쇄)
	요청을 처리할 수 있는 객체를 연결하는 패턴
-------------------------------------------------------------------------------------------
	Single 하나만 만든다
	Factory	생성을 위임
	Builder 단계적 생성
	Adapter	인터페이스 변환
	Facade	단순 창구
	Template 골격 정의
	Strategy 알고리즘 교체
-------------------------------------------------------------------------------------------

4. Wrapper Class
	기본 자료형들의 클래스 타입

	기본타입(원시타입)	wrapper 클래스(참조형타입)
	byte		Byte
	short		Short
	int		Integer
	long		Long
	double		Double
	float		Float
	boolean		Boolean
	char		Charater

1) 기본형(Primitive Type)
	첫글자 소문자
	null 초기화 불가능
	값을 그대로 저장한다

2) 참조형(Wrapper Class)
	첫글자 대문자
	Null 초기화 가능
	객체의 레퍼런스를 저장한다

+) 메모리를 더 많이 차지하는 Wrapper 클래스 언제 사용할까?
	기본형이 객체로 취급되어야하는 경우들이 있는데 이 때 Wrapper Class를 사용한다
	- 매개변수로 객체가 요구될 때
	- 기본형 값이 아닌 객체로 저장해야할 때
	- 객체간의 비교가 필요할 때

3) 박싱과 언박싱
	기본타입의 값을 포장 객체로 만드는 과정을 박싱(Boxing)이라고 하고
	포장 객체에서 기본 타입의 값을 얻어내느 과정을 언박싱(Unboxing)이라고 한다

4) boxing
- 예전 방법
	클래스타입 객체 = new 클래스타입(일반타입의 값);

- 최근 방법
	클래스타입 참조변수 = 클래스타입.valueOf(일반타입의 값);

- 자동 boxing
	클래스타입 참조변수 = 일반타입의값;

5) unboxing
	일반타입 변수 = 참조변수.000Value();

- 자동 unboxing
	일반타입 변수 = 참조변수;


6) Wrapper 클래스를 사용하는 이유
	원시타입(일반타입)을 박싱하면 다양한 메소드를 제공받을 수 있다
	여러 일반타입을 하나의 타입으로 묶기 위해서는 반드시 클래스타입(Wrapper Class)으로 사용해야하고 이 때 박싱해야된다
	
5. String 메소드
	charAt()	문자열에서 해당 인덱스 번호의 문자를 char타입으로 반환
			"hello".charAt(0)	h
	length()	문자열의 길이 반환
	indexOf()	특정 문자나 문자열을 앞에서부터 찾아 처음으로 찾게되는 인덱스를 반환
			없을경우 -1을 반환
	split()		구분문자를 기준으로 문자열을 잘라 배열에 담아 반환
	strip()		문자열 앞뒤 공백 제거
	substring(start, end)	시작인덱스 번호 ~ 끝 인덱스-1까지의 문자열을 반환

6. 컬렉션 프레임워크
1) 자료구조와 알고리즘
자료구조
	많은양의 데이터를 효율적으로 관리(추가, 삭제, 수정, 검색)하고자 데이터를 저장하는 방법
	자바에서는 여러 자료구조 알고리즘을 api로 제공하고 있기 때문에 복잡한 알고리즘을 몰라도 api를 사용하여 자료구조를 구현할 수 있다
	자료구조를 통해 데이터 그룹을 효과적으로 처리할 수 있도록 지원하는 자바 API를 자바 컬렉션 프레임워크라고 한다

알고리즘
	어떤 문제가 발생했을 때 해결할 수 있는 절차 혹은 순서

	java.util 패키지에 포함되어있다
