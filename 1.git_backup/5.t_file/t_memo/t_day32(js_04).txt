1. javascript 쓰레드
1) 동기화(synchronized)
	동기와 비동기의 흐름을 알기 위해 쓰레드 개념을 먼저 알아야한다
	프로그램, 프로세스 이해가 필수!!

- 프로그램(program)
	디스크에 저장된 실행 파일 그 자체
	실행되지 않은 상태
	메모리, CPU 자원 할당 없음
	단순한 코드 덩어리(명령어의 집합)
	ex) chrome.exe, node, main.exe	

- 프로세스(process)
	프로그램이 실행되어 메모리에 올라간 상태
	운영체제가 관리하는 실행 단위
	독립적인 메모리 공간을 가짐

	프로세스 메모리 구조
	Code 영역
	Data 영역
	Heap 영역
	Stack 영역
	-> 기본적으로 다른 프로세스와 메모리를 공유하지 않음
	(공유하려면 IPC, 공유 메모리 등이 필요함)

2) 쓰레드(Thread)
	프로세스 내부에서 CPU 작업을 수행하는 실행 단위
	비동기식 방식으로 특정 연산을 수행할 때 업무를 효율적으로 처리하기 위해 짧은 시간에
	다양한 연산들을 마치 동시에 처리하는 것처럼 할 수 있게 하는 것

(1) 싱글쓰레드
	처리 경로를 한개만 가지고 있기 때문에 직렬적이다
	동시에 많은 양을 처리하기는 힘들기 때문에 상대적으로 비효율적이다
	하지만 하나의 작업에 문제가 발생하더라도 다른 작업에는 영향을 끼치지 않는다
	따라서 안정성이 보장되고 설계 시 멀티 쓰레드에 비해 쉽다

(2) 멀티쓰레드
	하나의 프로세스를 동시에 처리하는 것처럼 보이지만 사실은 매우 짧은 단위로 분할해서 차례대로 처리한다
	여러개의 처리경로를 가질 수 있도록 하며, 동시작업도 가능해진다
	설계하기 굉장히 어렵고, 하나의 쓰레드가 문제 발생 시 모든 쓰레드에 문제가 발생한다
	처리량 증가, 효율성 증가, 처리비용 감소의 장점이 단점을 감수하고 설계한다

+) 논리적 동시성(Concurrency)
	싱글코어 CPU에서는 A -> B -> A -> B-> A -> B
	초고속 전환(Context Switching)
	사람 눈에는 동시에 보이나 실제로는 동시에 실행되지 않음

- 초고속 전환(Context Switching)
	CPU가 작업을 바꿀 때 
	- 현재 쓰레드 상태 저장 -> 다음 쓰레드 상태 복원 -> 실행 전환

+) 물리적 병렬성(Parallelism)
	멀티코어 CPU에서는 
	Core1 -> A
	Core2 -> B
	동시에 진짜 실행됨

(3) 자바스크립트의 쓰레드
	자바스크립트는 단일 쓰레드로 한번에 하나의 작업만 수행할 수 있다
	자바스크립트는 이벤트 루프를 활용하여 태스크가 종료되지 않아도 곧바로 실행하는 함수를 처리하는 방식을 도입했다
	동기의 코드는 순차적인 흐름이지만 비동기는 처리시간이 각각 다르기 때문에 결과를 예상할 수 없다
	따라서 결과에 비동기 이후 실행된 동기적 함수를 실행하기 위하여 비동기를 동기적으로 처리한다

+) 브라우저 전체 구조
	js 엔진과 브라우저 전체는 다르다
	브라우저는 멀티 프로세스, 멀티 쓰레드 환경이다

브라우저의 구성요소
(1) JS Engine (ex : V8)
	싱글 콜스택
	JS 코드 실행 담당

(2) Web APIs
	브라우저 제공하는 기능 : setTimeout, DOM 이벤트 처리, fetch, 네트워크, 파일 읽기
	브라우저 내부 C++코드 + 별도 쓰레드에서 처리된다

(3) Task Queue(Callback Queue)
	비동기 작업 완료 시 콜백이 이 공간에 들어간다

(4) Event Loop
	계속 반복
	- Call Stack이 비었는지 확인 -> 비었으면 Queue에서 하나 꺼냄 -> Stack 넣고 실행
	이 구조 덕분에 JS 싱글 쓰레드지만 비동기 처리가 가능한다

2. 동기와 비동기
1) 동기
	요청을 보낸 후 다음 응답을 받아야만 다음의 동작이 이루어지는 방식을 의미한다
	모든일은 순차적으로 실행되며 어떤 작업이 수행중이라면 다음 작업은 대기하게 된다

2) 비동기
	데이터를 수신하는 코드와 페이지를 표시하는 결과 비동기적으로 처리하는 방식을 의미한다
	들어오는 일은 모두 우선적으로 받아주고 끝났다는 이벤트가 전달되면 해당 이벤트를 처리한다
	자바스크립트에서는 기존에 콜백함수를 이용했지만 콜백 지옥에 빠질 가능성이 높기 때문에
	ES7이상에서는 Promise를, ES8이상에서는 async와 await을 지원한다

+) 동기 / 비동기 / 병렬 / 동시성
	동기	이전 작업이 끝날 때까지 대기
	비동기	기다리지 않고 다음 작업 진행
	병렬	물리적으로 동시에 실행
	동시성	동시에 실행되는 것처럼 보임


3. 프로미스(Promise)
	자바스크립트 비동기 처리에 사용되는 객체이다
	주로 서버에서 받아온 데이터를 화면에 표시할 때 사용한다

- 프로미스의 상태 3가지
	대기(pending)	비동기 작업이 아직 완료되지 않은 상태
	이행(fulfilled)	비동기 작업이 성공적으로 완료된 상태
	거부(rejected)	비동기 작업이 실패한 상태

4. async / await
	async 함수는 함수 선언 앞에 async 키워드를 붙여서 정의한다
	항상 Promise를 반환한다
	async 함수내에서 await 키워드를 사용하여 다른 프로미스나 기본값을 기다릴 수 있다
	
	await 키워드는 async 함수 내에서만 사용할 수 있다
	프로미스가 처리될 때까지 해당 함수의 실행이 일시중지된다
	프로미스가 이행되면 해당 값을 반환한다
	await 표현식은 프로미스를 반환하는 모든 비동기 작업에 사용될 수 있다

5. fetch
	자바스크립트에서 제공하는 비동기 네트워크 통신을 위한 기능이다
	HTTP 요청을 보내고 응답을 받아 사용한다
	웹 어플리케이션에서 서버와 데이터를 주고 받을 때 많이 사용된다

	fetch는 URL을 매개변수로 사용하여 요청을 보내고 프로미스를 반환한다
	네트워크 요청 : HTTP 요청 메소드(GET, POST, PUT, DELETE 등) 요청 헤더, 본문 등을 설정할 수 있다
	프로미스 기반 : .then(), .catch()를 사용하여 응답을 처리할 수 있다

1) fetch 특징
- Promise 기반 : fetch는 즉시 Promise를 반환한다
	네트워크 요청은 백그라운드에서 진행된다
	완료되면 Promise가 resolve된다

- 반환값은 데이터가 아니다
	response는 JSON이 아니라 Response 객체기 때문에 데이터를 꺼내기 위해서는 추가 변환이 필요하다
	
	fetch("url")
	.then(response => console.log(response); (x)
	
	fetch("url")
	.then(response => response.json())
	.then(date => console.log(data));


+) AJAX(Aynchronous Javascript and XML)
	페이지를 새로고침 하지 않고 서버와 데이터를 주고 받는 기술
	웹 페이지 전체를 다시 로딩하지 않아도 일부 데이터만 바꿀 수 있다
	초창기엔 XML을 썼지만 요즘은 JSON과 함께 사용됨
	AJAX는 기술이 아니라 하나의 방법론(과거 : XMLHttpRequest 객체를 이용해 구현)

	const xhr = new XMLHttpRequest();
	xhr.open("get", "https://jsonplaceholder.typicode.com/users");
	xhr.onreadystatechange = function() {
		if(xhr.readyState === 4 && xhr.status === 200){
			const data = JSON.parse(xhr.responseText);
			console.log(data);
		}
	};
	xhr.send();
		

+) fetch
	XMLHttpRequest를 대체하기 위해 만들어진 더 간단하고 강력한 함수(브라우저 내장함수, Promise 기반 동작)

	fetch("https://jsonplaceholder.typicode.com/users")
	.then((response) => response.json())
	.then((data) => console.log(data))
	.catch((error) => console.error(error))
