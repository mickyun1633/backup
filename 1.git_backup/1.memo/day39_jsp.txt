7) ContextPath와 servlet 경로

		보통 프로젝트명과 동일하거나 /

	Servlet 경로(Servlet Mapping Path) : 특정 servlet과 연결된 URL 경로(servlet 단위)
		사용자가 이 주소를 입력하면 해당 servlet이 실행된다

		어노테이션 방식

		ContextPath/output

		web.xml 밧익

			<servlet-naem>MyServlet</servlet-name>
			<url-pattern>/output</url-pattern>
		</servlet-mapping>

	server 주소 = ContextPath + Servlet 경로

		OutputServlet에서 setAttribute한 값,
		getParameter로 받은 값
	전부 그대로 유지한 채 jsp로 전달

forward 전체 흐름
	browaer 전송 버튼 -> /output 요청 (web.xml에서 /output 경로가 있는 지 확인 후)
	OutputServlet 실행 
	request에 데이터 저장
	forward(request, response)
	request 데이터 사용 가능

2. GET과 POST
1) GET
	데이터를 URL에 붙여서 전송

2) POST
	데이터를 요청 본문(body)에 담아서 전송
	URL에는 데이터가 보이지 않는다

3) servlet에서의 차이
- GET 요청
	protected void doGet(HttpServletRequest request, HttpServletResponse response)
- POST 요청


4) form에서 사용하는 방법
- GET 방식
	<form method="get" action="서블릿경로">
- POST 방식
 	<form method="post" action="서블릿경로">

5) 중요한 차이
- URL 노출 여부
	GET : 비밀번호 노출됨
	POST : URL에 안보임

- 한글 처리 여부
	POST는 반드시 필요함			request.setCharacter
	GET은 server 설정에 따라 다름

- 내부 동작 차이
	GET : Header + QueryString
	POST : Header + Body

+) 멱등성(idempotent)
	GET : 여러번 요청해도 결과 동일
	POST : 여러번 요청하면 데이터 중복 가능

- 요청 -> Servlet 경로(wed.xml) -> servlet 실행 -> get / post 실행 -> 응답
