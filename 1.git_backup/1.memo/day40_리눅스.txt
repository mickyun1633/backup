3) /etc/passwd
	사용자 계정의 기본 정보 저장 파일
	모든 사용자가 읽기(644)
	
	계정명:비밀번호:UID:GID:설명:홈디렉터리:로그인쉘
	로그인id => 로그인id
	비밀번호 => x
		실제 암호는 /etc/shadow
	UID	=> 사용자 고유번호
		0 root
		1-99 시스템 계정
		1000이상 일반사용자(배포판 따라 다를 수 있음)
	GID	=> 기본 그룹 번호
	설명	=> 사용자 정보(주석)
	홈디렉터리	=> 로그인 시 이동 위치
	로그인 쉘	=> /bin/bash, /sbin/nologin(로그인 불가 계정)

	text:x:1001:1001:Test User:/home/test:/bin/bash

* /etc/shadow
	사용자의 암호(해시값) 및 비밀번호 정책 저장

	root만 읽기 가능
	일반 권환 : 000 (암호해시를 보호하기 위함)

- 파일형식
	계정명:암호:최근변경일:최소사용일:최대사용일:경고일:유예일:계정만료일:예약

	계정명	passwd 파일의 계정과 동일
	암호	암호화 해시값
		$1$	MD5
		$2a$	Blowfish
		$5$	SHA-256
		$6$	SHA-512(현재 기본)
		
		특수기호 차이
		!!	비밀번호 미설정
		!	계정 잠금
		·	로그인 불가 계정

	최근병결일 1970년 1월 1일부터 경과한 일수
	최소사용일		이 기간 전에는 비밀번호 변경 불가
		0	제한없음
	최대사용일		이 일수가 지나면 변경해야함
		90	90일 후 만료
		99999	사실 무제한
	경고일		만료 며칠 전 경고할 지
	유예일		만료후 며칠동안 로그인 허용할 지
	계정만료일		이 날짜가 지나면 계정 자체가 만료
	예약필드		보통 비어있음

비밀번호 정책 ★★★★★
chage		비밀번호 만료 관리
	-l user 설정 확인
	-d 날짜 	마지막 변경일
	-m	최소사용일
	-M	최대사용일
	-w	경고일
	-E	계정 만료일

	ex) 
	비밀번호 최대 사용기간 90일
	chage -M 90 test
	chage -m 90
	change -M	=> x

chmod	파일 / 디렉터리 권한 변경
	chmod [옵션] 권한 파일명

문자방식
	chmod	[u/g/o][+/-][rwx] 파일명
	chmod	u+rw 파일명

	-R 하위 디렉터리까지 재귀적 사용

숫자방식
		U     G    O
	chmod	[421][421][421] 파일명
		777
		431

특수권한
	SUID = 4000
	SGID = 2000
	Sticky = 1000

	chmod 4755 의미 => SUID O, U - 모든권한, G - 읽기, 실행권한, O - 읽기, 실행권한 가능
