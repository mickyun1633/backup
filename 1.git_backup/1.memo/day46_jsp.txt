4) MyBatis 관련 기술
(1) DBCP(DataBase Connection Pool)
   문제 : DB 연결은 비용이 크다
   사용자 요청 -> DB연결 -> 인증 -> 작업 -> 연결 종료
   해결 : Connection Pool
      DB연결을 미리 여러개 만들어놓고 보관
      필요할 때 빌려주고 다시 반환한다
   요청 -> Pool에서 Connection 꺼냄 -> SQL 실행 -> 사용 후 Pool에 반납

(2) JNDI(Java Naming and Directory Interface)
   외부 자원을 이름으로 찾아서 사용하는 기술
   외부자원 : DB연결정보(드라이버, URL, username, password), DBCP, 환결설정, 메일서버

   서버에 등록 -> 이름으로 찾아 사용

(3) DBCP와 JNDI 관계
   DBCP      Connection Pool 관리
   JNDI      서버에 등록된 DataSource를 이름으로 찾아 사용

5) MyBatis 동작구조

6) Mapper.xml에서 id를 사용하는 이유
   MyBatis에서 SQL은 Mapper.xml 안에 여러 개 작성된다

   만약 insert SQL이 여러개 있다면 MyBatis가 어떤 sql을 실행해야하는지 구분해야한다
   그래서 사용하는 것이 id이다
   id => SQL을 구분하는 고유한 이름

- SQL 호출 방식
   Mapper이름(namespace) + id 이름
   sqlSession.insert("users.join", userMap);
   users.join => users.xml 안에 있는 join SQL문을 실행

- JDBC 방식의 문제
   "insert into tbl_user values(seq_user.nextval, ?, ?, ?)"

   ps.setString(1, userId);
   코드가 길다
   sql과 java코드가 섞인다
   가독성이 떨어진다

- MyBatis 값 전달 방식
   #{} 문법을 사용한다
   insert into tbl_user(user_number, user_name, user_id, user_pw)
   values(seq_user.nextval, #{userName}, #{userId}, #{userPw})

   동작 : map.get("userName")    
      값을 자동으로 넣어준다


- parameterType
   Mapper.xml은 외부에서 어떤 데이터 타입이 들어오는지 모른다
   SQL 태그에 parameterType을 지정한다
   <insert id="join" parameterType="map">   #외부에서 map 객체가 전달된다

   #{userId}   -> map.get("userId")로 자동 매핑된다

- parameterType 기본 타입
   소문자를 사용한다
   종류    타입
   문자열    string
   정수    int, _int
   실수    double, float
   논리    boolean
   Map    map
   객체    DTO
   컬렉션    list, collection
   배열    array
   날짜    date
   PK시퀀스 -> Long 사용이 안전함

2. MVC
	Model : DAO, DTO/VO, mapper, DB
		어플리케이션의 데이터를 처리하고 DB에 접근하며 비즈니스 로직과 데이터를 담당

	View : jsp
		사용자 인터페이스(UI)를 담당
	Controller : FrontController, Controller
		어플리케이션의 흐름을 제어하는 역활을 담당

+) DTO와 VO
	DTO(Data Transfer Object)
		데이터 전송를 위해 사용
		변경 가능, 많은 필드 포함, getter / setter 제공
	
	VO(Value Object)
		테이터의 의미를 표현하기 위해 사용
		불변, 몇 가지 필드만 포함, 생성자에게서 초기화











async function loadLayout() {
    const isLoggedIn = localStorage.getItem('isLoggedIn') === 'true';

    const rootPath = './../../'; 
    const headerFile = isLoggedIn 
        ? `${rootPath}html/components/header_login.html` 
        : `${rootPath}html/components/header_logout.html`;
    const footerFile = `${rootPath}html/components/footer.html`;

    try {
        const hResp = await fetch(headerFile);
        if (!hResp.ok) throw new Error(`헤더 없음: ${headerFile}`);
        const hData = await hResp.text();
        document.getElementById('header-container').innerHTML = hData;
    } catch (e) {
        console.error("헤더 로드 실패", e);
    }

    try {
        const fResp = await fetch(footerFile);
        if (!fResp.ok) throw new Error(`푸터 없음: ${footerFile}`);
        const fData = await fResp.text();
        document.getElementById('footer-container').innerHTML = fData;
    } catch (e) {
        console.error("푸터 로드 실패", e);
    }
}

window.addEventListener('DOMContentLoaded', loadLayout);
