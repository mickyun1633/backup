5. 배열
	javascript의 배열은 일반 배열처럼 보이고 배열처럼 사용하지만 리스트에 가깝다
	배열이 저장하는 값을 요소(element)라고 부르며 요소로 저장하는 값은 원시 타입 뿐만 아니라 객체 타입도 가능하다

1) 배열 생성

	let ar2 = [ , , , ];

	초기값 할당
	let ar3 = new Array(1, 2, 3);

	배열 크기 지정
	let ar4 = new Array(3); // 요소가 1개이고 숫자인 경우 배열 크기로 사용된다

4) 배열에서 사용하는 반복문
- for ... in
	객체의 열거가능한 속성을 순회할 때 주로 사용
	배열이나 객체의 key값을 순회
	배열의 각 요소(값)을 순서대로 순회한다

- for ... of
	이터러블 객체(반복가능한 객체, 배열/문자열/Map/Set 등)를 순회할 때 주로 사용
   	value(배열의 값)을 순회
   	배열의 각 요소(값)을 순서대로 순회한다

6. map()함수
	배열의 각 요소에 대해 주어진 콜백함수를 호출하고 각 요소를 새로운 배열로 매핑시킨다
	const newArray = array.map(callback(element [, index [, array]][, thisArgs]));

	element	: 현재 처리중인 요소의 값
	index : 현재 처리중인 요소의 인댁스(선택)
	array : map()을 호출한 배열(선택)
	thisArgs : callback 함수 내에서 this 키워드가 참조할 객체(선택)

7. filter() 함수
	주어진 콜백함수로 만든 배열 요소를 테스트하고 조건을 충족하는 요소만을 가지고 새로운 배열을 생성한다
   	const newArray = array.filter(callback(element [, index [, array]][, thisArgs]));

   	element : 현재 처리중인 요소의 값
   	index : 현재 처리중인 요소의 인덱스(선택)
   	array : map()을 호출한 배열(선택)
   	thisArgs : callback 함수 내에서 this 키워드가 참조할 객체(선택)  

8. Javascript 엔진 구조와 동작원리
1) Javascript 엔진 구조
	싱글 쓰레드(Single-Thread)언어이다
	한번에 하나의 작업만 실행할 수 있으며 동시에 여러 작업을 직접 처리하지 못한다
	비동기 처리를 총해 동시성(Concurrency)을 지원한다

+) 싱글 쓰레드(Single-Thread)
	한번에 하나의 작업만 처리할 수 있는 구조를 의미한다
	javascript 엔진은 단일 쓰레드에서 코드를 실행하며 동시에 여러 작업을 실행할 수 없다
	즉, 코드가 순차적으로 실행된다

+) 동시성 모델
	콜스택(call stack)		현재 실행중인 작업(LIFO)
	태스크 큐(Task Queue)		비동기 작업의 콜백함수 대기열
	이벤트 루프(Event Loop)		콜스택과 태스크 큐를 관리하며 작업을 순차적으로 수행

+) 이벤트 루프의 작동방식 
	콜스택에 작업이 있는지 확인 : 스텍이 비어있지 않으면 현재 작업을 계속 실행
	콜스택이 비어있으면 	    : 태스크 규에서 대기중인 작업을 가져와서 실행
	비동기작업 		    : 태스크 큐에 콜백함수 형태로 저장   

2) Javascript 엔진의 구성요소
- 콜스택(Call Stack)
	함수 호출 순서를 관리하는 LIFO(Last-In First-Out) 구조의 스택
	실행중인 함수가 위로 쌓이고(push) 실행 완료되면 제거(pop)

	function a() {
		b();
	}
	function b() {
		console.log("실행");
	}

	a(); => main() -> a() -> b() -> console.log();


- 힙(Heap)
	객체, 배열, 함수 등 참조형 데이터가 저장되는 동적 메모리 영역
	크키가 정해지지 않고 필요헤 따라 공산을 할당하거나 해제한다
	변수는 힙의 주소(참조값)을 Stack에 보관하고 실제 데이터는 Heap에 저장된다
   
   	let user = {name : "짱구"};
   
  	Stack : user -> Heap 주소
   	Heap : {name : "짱구"} 

- 태스크 큐(Task Queue / Callback Queue)
	비동기 작업의 콜백함수가 대기하는 공간
	
	대표적인 task : setTimeout, setInterval, DOM 이벤트, I/O 콜백

   	console.log("시작");
   	setTimeout(() => {console.log("1초뒤 실행");}, 1000);
   	console.log("끝");

   	시작 => 콜 스택
   	setTimeout -> WebAPI 이동
   	끝 => 콜 스택
   	1초 후 콜백 -> Task Queue
   	콜 스택이 비면 -> 실행

   	시작
   	끝
   	1초뒤 실행

+) 태스크 큐의 종류
  	 매크로 태스크(Task Queue)      setTimeout, 이벤트
   	마이크로 태스크(Microtask Queue)   Promise.then, async/aswait

   	* 마이크로 태스크가 항상 우선 실행!!

   	setTimeout(() => console.log("timeout"));
   	Promise.resolve().then(() => console.log("promise"));

   	promise
   	timeout

3) 이벤트 루프 구조
   
   	Call Stack      현재 실행중인 동기 코드
    	   ▲
           ▼
   	Task Queue      비동기 콜백함수 대기(setTimeout, 이벤트 등)
       	   ▲
           ▼
   	Event Loop      콜 스택이 비면 큐에서 꺼내 실행(스케줄링 관리자)

- 동작원리
   	(1) 콜스택에 실행중인 코드가 있는지 확인
   	(2) 비어있으면
     	Microtask Queue 먼저 전부 실행
      	그 이후 Task Queue에서 하나 실행
   	(3) 다시 1번으로 반복
	
	=> 싱글 쓰레드 + 비동기 처리 + 순차 실행이 가능함

4) 자바스크립트 메모리구조(RAM 기준)

=====================================

---------------------------------------

---------------------------------------
Data영역         전역변수, 초기화된 정적 데이터
---------------------------------------
Text영역         코드(명령어) 자체가 저장됨

	객체 사용 : .	['key']
	배열 사용 : 배열명[idx]

6) 반복문
	for ... in	: 객체, 배열
	for ... of	: 배열, map, set 사용가능

7) map() 함수
	배열의 각 요소에 대해 주어진 콜백함수를 호출하고 각 요소를 새로운 배열로 매핑시킨다
	const newArray = array.map(callback(element [, index [, array]][, thisArgs]);

8) filter() 함수
	주어진 콜백함수로 모든 배열 요소의 조건을 충족하는 요소만들 가지고 새로운 배열을 생성한다
	const newArray = array.filter(callback(element) [, index [, array]][, thisArgs]);

	element	: 현재 처리중인 요소의 값
	index	: 현재 처리중인 요소의 인덱스(선택)
	array	: map(), filter()를 호출한 배열(선택)
	thisArgs: callback 함수 내에서 this 키워드가 참조할 객체(선택)

9) reduce() 함수
	배열을 순회하면서 하나의 값으로 누적할 때 사용하는 함수
	array.reduce(callback, initalValue);

   	callback : 배열의 각 요소에 대해 실행할 함수
   	initalValue : 누적 시작 값, 첫번째 호출에서 사용될 값(작성하지 않으면 배열의 첫번째 요소가 initalValue로 사용되고
      		두번째 요소부터 순회가 시작된다)
   
   	accumulator : 누적된 값으로 각 호출에서 callback 함수가 반환된 값
   	currentValue : 현재 처리중인 배열의 요소
  	currentIndex : 현재 요소의 인덱스(선택)
   	array : reduce()가 호출된 원본 배열(선택)
