1. 전체흐름 파악
	ConsoleUI가 메뉴를 보여줌
	사용자가 로그인(LoginManager)
	로그인 성공하면 LibraryService를 통해
		도서 목록 보기/ 검색 / 대출 / 반납
	대출 / 반납 과정에서 Book(Borrow interface 구현) + Member(User 상속)
	조건 위반 시 사용자 예외 발생

2. 역활별 어떤 책임만 갖게 할지(SRP)
	ConsoleUI	: 입력/출력만 담당
	LibraryService 	: 책 찾기 / 대출 / 반납 처리 담당
	LoginManager	: 사용자 로그인 처리 담당
	Member/User 	: 내 정보 + 내가 빌린 책 목록 상태 담당
	Book		: 책 정보 + 대출가능 상태 담당
	예외		: 예외 발생하는 것을 메시지 보여줌 

3. 클래스 설계(어떤 필드/메소드 가져가야하는지)
1번 	User 추상클래스 + LoginManager
	User(abstract)
		필드(기본정보) : name, id, pw
		생성자(기본생성자, 매개변수 생성자)
		메소드
		getter메소드 : getter만, sette(pw)
		추상메소드	   : boolean login(Stirng id, String pw);
 
	LoginManager	
		필드(private) : Map<String, user> userMap
		메소드
		userMap 사용자 추가 : addUser(User user)
		사용자 조회		 : findUser(String userId)
		사용자 로그인 	 : login(String id, String pw)
					userMap에서 찾고 없으면 로그인 실패
					있으면 결과 반환

		메소드
			도서대여 메소드	     => 30권 초과면 예외, 정상이면 추가
			도서반납 메소드	     =>	목록에 존재하는지 체크(없으면 false)
			대출중인 책 목록 확인 메소드 => 내가 빌린 책인지 확인
