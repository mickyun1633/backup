0. 복습 
1) 이상현상 	
	dataBase 설계까 잘못되어 발생하는 문제
	- 삽입이상
	- 갱신이상
	- 삭제이상

2) 정규화(Normalization)
	삽입/갱신/삭제 이상현상을 제거하기 위한 작업
	data 중복을 최소하하는 것이 목적

- 1차 정규화
	table의 모든 column의 값은 원자값을 가져야한다
	반복되는 그룹을 별도의 table로 분리해야한다

- 2차 정규화
	1차 정규화를 막족해야한다
	기본키가 복합키일 경우 부분함수 종속 제거해야한다

- 3차 정규화
	이행적 함수를 제거하여 table을 분리한다

3) 집계함수
	그룹화된 data를 집계 연산을 통해 하나의 결과(행)로 얻는다
	
	COUNT(*)
	COUNT(column)
	SUM()
	AVG()
	MAX(), MIN()

4) SELECT문
	SELECT [DISTINCT] column명 [AS 별칭]
	FROM table명
	WHERE 조건식
	GROUP BY 그릅화 기준 column
	HAVING 그룹 조건식
	ORDER BY 졍렬기준column [ASC/DESC];

	작성순서 : SELECT -> FROM -> WHERE -> GROUP BY -> HAVING -> ORDER BY
	실행순서 : FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> OREDR BY

5) ROWNUM
	결과 행 앞에 1부터 1씩 증가하는 시퀀스를 붙여준다
	ROWNUM은 column처럼 사용하면 된다
	*를 다른 column과 함께 조회하게 되면 소속(어디 테이블인지)을 명시해야한다
	table에서 ALIAS를 부여하고 해당 별칭은 * 앞에 명시해준다(별칭, *)

1. JOIN
	두 개 이상의 table을 연결하여 data를 검색하는 방법
	여러 table에 흩어져 있는 정보 중 사용자가 필요한 정보만 가져와서 가상의 table처럼 만들고 결과를 보여주는 것

	정규화를 통해 조회 table이 너무 많이 쪼개져있으면 작업이 불편하기 때문에 입력, 수정, 삭제의 성능을 향상시키기 위해서
	JOIN을 통해 합친 후 사용한다

★★★★★ SQL 실행순서
	FROM -> ON -> JOIN -> WHERE -> GROUP BY -> HAVING => SELECT -> ORDER BY

	FROM : table이나 view의 data를 가져오기
	ON : JOIN절의 조건 평가
	JOIN : 지정된 조건에 따라 두 개 이상의 table을 JOIN(합치기)
	WHERE : 조건에 맞는 행을 필터링
	GROUP BY : data 그룹화(집계함수와 주로 사용)
	HAVING : GROUP BY로 그룹화된 data에 대한 조건 적용
	SELECT : 조회할 column을 선택(중복제거, 집계함수, 별칭)
	ORDER BY : 최종 결과를 정렬(별칭, ASC/DESC)


1) JOIN 종류
	내부 조인	공통 존재 column의 값이 같은 경우를 추출하는 기법

	외부 조인	왼쪽 외부조인, 오른쪽 외부조인, 완전 외부조인
		- 왼쪽 외부조인 : 왼쪽 table의 모든 data와 오른쪽 table의 동일 data를 추출하는 기법
		- 오른쪽 외부주인 :오른쪽 table의 모든 data와 왼쪽 table의 동일 data를 추출하는 기법
		- 완전 외부조인 : 양쪽의 모든 data를 추출하는 기법

	교차 조인	조인 조건이 없는 모든 data 조합을 추출하는 기법

	셀프 조인	자기자신에게 별칭을 지정한 후 다시 조인하는 기법

2) 내부조인
	조건이 정확히 일치하는 값만 합쳐서 조회
	
	SELECT A.column1, A.column2
	FROM table1 A [INNER] JOIN table2 B
	ON [조인조건] -- A.column명1 = B.column명1
	[WHERE 검색조건];

	ex)

	[도서]		[도서가격]

	2000	자료구조	2000	25000
	3000	컴퓨터구조	3000	15000
			4000	20000

- 등가조인 
	두 table간의 column의 값이 서로 일치할 때 ON절에 등호(=)가 있을 때
	일반적으로 PK <-> FK 관계를 이용하여 조인을 주로 사용한다
	두 table간의 관계가 있다면 (FK가 존재한다면) 부모 table의 PK와 자식 table의 FK를 등호로 비교가 가능하기 때문에
	등가조인 사용이 가능하다

- 비등가조인
	두 table의 특징 column이 일치하지 않는 경우, ON절에 등호(=)가 없을 때	
	= 연산자가 아닌 BETWEEN, >, >=, <, <= 등의 연산자를 사용한다

3) 외부조인(OUTER JOIN)
	내부조인과는 다르게 한쪽에만 값이 있어도 table을 합쳐서 보여준다

- 왼쪽 외부조인 => 왼쪽 table 기준 / 오른쪽 table에 일치하는 column이 없을 때는 오른쪽의 column은 NULL로 채워진다

	SELECT A.column1, A.column2, B.column1
	FROM table1 A LEFT [OUTER] JOIN table2 B
	ON 조인조건
	[WHERE 검색조건];

- 오르쪽 외부조인 => 오른쪽 table 기준 / 왼쪽 table에 일치하는 column이 없을 때는 왼쪽의 column값은 NULL로 채워진다

	SELECT A.column1, A.column2, B.column1
	FROM table1 A RIGHT [OUTER] JOIN table2 B
	ON 조인조건
	[WHERE 검색조건];

- 완전 외부조인 => 양쪽 모든 값을 기준 / 모든 column의 값이 다 나온다
		없는 column은 NULL로 채워짐

	SELECT A.column1, A.column2, B.column1
	FROM table1 A PULL [OUTER] JOIN table2 B
	ON 조인조건
	[WHERE 검색조건];

2. 집합
	UNION : 합집합, 중복을 허용하지 않는다 (중복제거)
	UNION ALL : 합집합, 중복을 허용한다
	INTERSECT : 교집합
	MINUS : 차집합

1) 집합연산자가 있는 이유
	dataBase에서 여러 퀴리의 결과를 조합하고 특정 방식으로 결과를 출력하기 위해 존재한다
	여러 table이나 쿼리의 결과를 효과적으로 비교, 병합, 차 구하기 등으로 처리해야하는 data의 요구사항 때문에 존재한다
	
		집합연산 					WHERE 조건

쿼리실행방식	각 SELECT 쿼리가 독립적으로 실행		단일 쿼리로 실행

결과 처리 단계	개별 쿼리 결과를 결합 비교			조건 필터링만 수행

중복 제거 여부	UNION 중복제거, UNION ALL 중복제거 X		중복 제거는 따로 수행하지 않음

data source	여러table, 쿼리 조합 가능			단일 table에서 작동

성능		SORT, 비교 등의 추가 단계 느릴릴수 있음		단일 조건 처리로 비교적 빠름

사용 목적		독립된 결과를 병합하거나 차집합, 교집합		단일 table에서 특징 조건만 핉터링
		구할 때 사용

2) 집합 연산을 써야만 하는 상황(WHERE절로는 안되는 상황)
- 서로 다른 table끼리 비교
	EMPtable과 DEPTtable -> 하나의 WHERE 조건으로는 합칠 수 없음
	
- 조건이 겹치거나 복잡할 때
	SELECT * FROM EMP
	WHERE (SAL BETWEEN 1000 AND 2000) OR (SAL BETWEEN 10000 AND 30000);
	
	SELECT * FROM EMP WHERE SAL BETWEEN 1000 AND 2000
	UNION
	SELECT * FROM EMP WHERE SAL BETWEEN 10000 AND 30000

+) 활용
	상황					이유
	
	서로 다른 조건 결과를 한번에 출력하고 싶을 때	조건이 복잡하거나 명확한 구분이 필요한 경우가 있기 때문	UNION, UNION ALL	
	
	서로 다른 table의 결과 병합			조인 대신 간단히 병합할 때 				UNION
	
	중복 data도 모두 보고싶을 때			성능이 좋고 확인하기 쉽기 때문에			UNION ALL
	
	공통 data만 추출할 때 			두 조건을 동시에 만족하는 경우 				INTERSECT
	
	하나의 쿼리 결과에서 다른 쿼리 결과를 빼야할 때 	제외 조건을 명확하게 할 수 있기 때문 			MINUS

3. 서브쿼리(sub query)
	하나의 쿼리 내에 작성하는 또 다른 쿼리
	서브쿼리의 위치에 따른 종류
		- FROM절 : INLINE VIEW
		- SELECT절 : SCALAR
		- WHERE절 : SUB QUERY

- 인라인 뷰
	FROM 절 안에 또 다른 쿼리문을 작성한다
	서브쿼리의 결과를 가상 table처럼 사용한다

- 스칼라 서브쿼리
	SELECT 절 안에 또 다른 쿼리문을 작성한다
	단일컬럼, 단일 행을 반환한다(1개의 값으로 사용)

- 서브쿼리
	WHERE 절 또는 HAVING 절안에 또 다른 쿼리를 작성한다

연관 서브쿼리(Correlated Subquery)
	서브쿼리가 메인 쿼리의 컬럼을 참조한다
	메인쿼리의 각 행마다 반복 실행된다

비연관 서브쿼리(Non-Correlated Subquery)
	서븤뭐리가 메인 쿼리의 컬럼을 사용하지 않는다
	메인쿼리와 독립적으로 실행가능하다
	일반적으로 한 번만 실행된다

단일 행 서브 쿼리(Single-Row Subquery)
	하나의 값만 반환
	연산자 : =, >, <, >=, <=, <>(!=)

다중형 서브쿼리(Multi-Row Subquery)
 	여러 행(값) 반환
   	연산자 : IN, ANY, ALL, EXISTS, SOME

* 서브쿼리 : SQL안에 들어있는 SQL
	SELCET .. 
	FORM ..
	WHERE 컬럼 = (SELECT ..);
	
- 일반 서브쿼리
	안쪽실행 -> 결과 저장 -> 바깥 실행
