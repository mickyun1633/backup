1. Javascript 쓰레드
1) 동기화(synchronized)
	동기와 비동기 흐름을 알기 위해 쓰레드 개념을 먼저 알아야한다
	program, prose 이해가 필수!!

- program
	disk에 저장된 실행 file 그 자체
	실행되지 않은 상태
	memory, CPU 지워너 할당 없음
	단순한 code 덩어리(명령어의 집합)
	ex) chrome.exe, noed, main.exe

- process
	program이 실행되어 memory에 올라간 상태	
	OS가 관리하는 실행 단위
	독립적인 memory 공간을 가짐

	process memory 구조
	Code 영역
	Data 영역
	Heap 영역
	Stack 영역
	-> 기본적으로 다른 proces와 memory를 공유하지 않음
	(공유하려면 IPC, 공유 memory 등이 필요함)

2) 쓰레드(Thread)
	process 내부에서 CPU 작업을 수행하는 실행 단위
	비동기 방식으로 특정 연산을 수행할 때 업무를 효율적으로 처리하기 위해 짧은 시간에 
	다양한 연산들을 마치 동시에 처리하는 것처럼 할 수 있게 하는 것

(1) single thread
	처리 경로를 한개만 가지고 있기 때문에 직렬적이다
	동시에 많은 양을 처리하기에는 힘들기 때문에 상태적으로 비효율적이다
	하지만 하나의 작업에 문제가 발생하더라도 다른 적업에는 영향을 끼치지 않는다
	따라서 안정성이 보장되고 설계 multi therad에 비해 쉽다

(2) Multi thread
	하나의 process를 동시에 처리하는 것처럼 보이지만 사실은 매우 짧은 다위로 분활해서 차례대로 처리한다
	여러개의 처리경로를 가질 수 있도록 하며, 동시작업도 가능해진다
	설계하기 굉장히 어렵고, 하나의 thread가 문제 발생 시 모든 thread에 문제가 발생한다
	처리량이 중가, 효율성 증가, 처리비용 감소의 장점이 단점을 감수하고 설계한다

+) 논리적 동시성(Concurrency)
	single core CPU에서는 A -> B -> A -> B -> A -> B 
	초고속 전환(Context Switching)
	사람 눈에는 동시에 보이나 실제로는 동시에 실행되지 않음

- 초고속 전환(Context Switching)
	CPU가 작업을 바꿀 때
	- 현재 thread 상태 저장 -> 다음 thread 상태 복원 -> 실행 전환

+) 물리적 병렬성(Parallelism)
	multi core CPU에서는
	Core1 -> A
	Core2 -> B
	동시에 진짜 실행됨

(3) Javascript thread
	javascript는 단일 thread로 한번에 하나의 작업만 수행할 수 있다
	javascript는 event loop를 활용하여 task가 종료되지 않아도 곧바로 실행되는 함수를 처리하는 방식을 도입했다

	동기의 code는 순차적인 흐름이지만 비동기 처리시간이 각각 다르기 때문에 결과를 예상할 수 없다
	따라서 결과에 비동기 이후 실행된 동기적 함수를 실행하기 위하여 비동기를 동기적으로 처리한다

+) browser 전체 구조
	js 엔진과 browser 전체는 다른다
	browser는 multi proces, multi thread 환경이다

browser의 구성요소
(1) JS Engine (ex : V8)
	single call stack
	JS code 싱행 담당

(2) Web APIs
	browser 제공하는 기능 : setTimeout, DOM event 처리, fetch, network, file 읽기
	browser 내부 C++ code + 별도 thread에서 처리된다

(3) Task Queue(Callback Queue)
	비동기 작업 완료 시 callback이 이 공간에 들어간다

(4) Event Loop
	계속 반복
	- Call Stack이 비었는지 확인 -> 비었으면 Queue에서 하나 꺼냄 -> Stack 넣고 실행
	이 구조 덕분에 JS single thread지만 비동기 처리가 가능하다

2. 동기와 비동기
1) 동기
	요청을 보낸 후 다음 응답을 받아야만 다음의 동작이 이루어지는 방식을 의마한다
	모든일은 순차적으로 실행되며 어떤 작업이 수행중이라면 다음 작업은 대기하게 된다

2) 비동기
	data를 수신하는 code와 page를 표시하는 결과 비동기적으로 처리하는 방식을 의미한다
	들어오는 일은 모두 우선적으로 받아주고 끝났다는 event가 전달되면 해당 event를 처리한다
	javascript에서는 기존에 callback함수를 이용했지만 callback hell에 빠질 가능성이 높기 때문에
	ES7이상에서는 Promise를, ES8이상에서는 async와 await을 지원한다

+) 동기 / 비동기 / 병령 / 동시성 ★★★★★

	동기 	이전 작업이 끝날 때까지 대기

	비동기 	기다리지 않고 다음 작업 진행

	병렬 	물리적으로 동시에 실행
	
	동시성 	동시에 실행되는 것처럼 보임

3. Promise
	javascript 비동기 처리에 사용되는 객체이다
	주로 server에서 받아온 data를 화면에 표시할 때 사용한다

- Promise의 상태 3가지
	대기(pending)		비동기 작업이 아직 완료되지 않은 상태
	이행(fulfilled)		비동기 작업이 성공적으로 완료된 상태
	거부(rejected)		비동기 작업이 실패한 상태


4. async / await
	async 함수는 함수 선언 앞에 async keyword를 붙여서 정의한다
	항상 Promise를 반환한다
	async 함수내에서 awit keyword를 사용하여 다른 promise나 기본값을 기다릴 수 있다

	await keyword는 async 함수 내에서만 사용할 수 있다
	promise가 처리될 때까지 해당 함수의 실행이 일시중지된다
	promise가 이행되면 해당 값을 반환한다
	await 표현식은 promise를 반환하는 모든 비동기 작업에 사용될 수 있다

5. fetch
	javascript에서 제공하는 비동기 network 통신을 위한 기능이다
	HTTP 요청을 보내고 응답을 받아 사용한다
	웹 어플리케이션에서 server와 data를 주고 받을 때 많이 사용된다

	fetch는 URL을 매개변수로 사용하여 요청을 보내고 promise를 반환한다
	network 요청 : HTTP 요청 메소드(GET, POST, PUT, DELETE 등) 요청 헤더, 본문 등을 설정할 수 있다
	promise 기반 : .then(), .catch()를 사용하여 응답을 처리할 수 있다

1) fetch 특징
-Promise 기반 : fetch는 즉시 Promise를 반환한다
	network 요청은 백그라운드에서 진행된다
	완료되면 Promise가 resolve된다

- 반환값은 data가 아니다
	response는 JSON이 아니라 Response 객체기 때문에 데이터를 꺼내기 위해서는 추가 변환이 필요하다
	
	fetch("url")
	.then(response => console.log(response); (x)
	
	- get 요청
	fetch("url")
	.then(response => response.json())
	.then(date => console.log(data))
