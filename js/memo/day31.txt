1. DOM ★★★★★★★★★★
1) DOM(Document Object Model)
	html tag들을 하나씪 객체화 한 것을 의미한다
	html page 내용과 모양을 제어하기 위해서 사용하는 객체이며 html tag당 DOM 객체가 하나씩 생성된다
	html tag의 포함관계에 따라서 부모, 자식, 형제 관계로 구성된다
	html 요소 (<h1>, <div>, <input>)을 모두 노드(node)라는 객체로 구조화한다

2) DOM API
	DOM을 다루기 위한 programming interface(메소드, 속성 등)
	browser가 제공하는 Javascript 객체/메소드 집합
	ex) document.getElementById(), element.innerHTML 등

2. DOM Tree 구조
	문서 구조를 트리(Tree) 형태로 표현한 것
	최상위 노드 : document
	주요 노드 : 요소노드(Element Node) - HTML tag
		속성노드(Attribute Node) - tag 속성
		텍스트노드(Text Node) - text contents

   	<body>
     	   <div id="box">div box</div>
   	</body>

   	documen		browser가 제공하는 최상위 객체
   	|_ html         요소노드
           |_ body      요소노드
	       |_ div (id="box")	div 요소 노드, id="box" div 요소의 속성노드로 존재
	       |_ div box		div의 텍스트 노드


	<body>
	  <ul id="menu">
	    <li class="item">home</li>
	    <li class="item">about</li>
	  </ul>
	</body>


	 document
  	 |_ html
	    |_ body
		|_ ul (id="menu")
		    |_ li (class="item")
		    |   |_ home 텍스트노드
		    |_ li (class="item")
			|_ about 텍스트노드

4. DOM 요소 조작
	요소의 내용을 조작할 때 사용하는 속성

1) innerHTML
	요소의 내부 HTML을 나타내는 속성
	HTML tag를 포함한 요소의 내용을 변경할 수 있다
	ex) const test = document.getElementById("test");
		test.innerHTML = "<strong>안녕</strong>";

2) innerText
	요소의 text 내용을 나타내는 속성
	HTML tag를 고려하지 않고 text 내용만 조작할 수 있다
	ex) const test = document.getElementById("test");
		test.innetText = "<strong>안녕</strong>"; => 결과 <strong>안녕</strong>

3) 속성 조작
   	getAttribute('속성명')		속성 가져오기
   	setAttribute('속성명', '값')	속성 설정하기
   	removeAttribute('속성명')		속성 삭제하기

4) 스타일 조작
   	element.style.속성명="값"
      	속성명 : 카멜표기법으로 사용한다

5. Event
	발생한 사건을 의미한다
	사용자가 button을 클릭하거나 항목을 선택하는 등의 사건을 event라고 한다
	event가 발생하면 javascript의 code가 실행되도록 구현하여 사용자와 상호작용하는 page를 만들 수 있다

- Mouse Event
	click : 요소를 클릭하면 발생
	mouseover : 마우스 커서를 해당 요소에 올리면 발생
	mouseout : 마우스 커서가 해당 요소의 영역을 벗어나면 발생
	...

- Keybord Event
	keydown  : 키보드를 누르는 동안 발생
	keyup	 : 키보드를 눌렀다가 땔 때 발생
	keypress : 키보드를 눌렀을 때 발생
	..

- Web page loaging Event
	load   : 문서(web page)가 로딩이 끝나면 발생
	unload : 문서를 벗어날 때 발생
	scroll : 문서가 스크롤되면 발생
	.. 

- From Event
	입력이 가능한 요소의 이벤트
	focus  : from 요소에 focus되면 발생
	blur   : focus를 잃으면 발생
	change : 입력값이 바뀌거나 체크상태가 변경되면 발생
	submit : submit 버튼을 누르면 발생
	..

+) 이벤트를 처리하기 위해 onXXX속성을 사용한다
	onClick 속성은 이 요소에 clickEvent가 발생하면 실행된다
	속성값안에는 javascript 명령어를 작성한다

8. defer, async
	web browser에서 들어가는 script는 대부분 HTML보다 무겁다
	용량이 커서 download 받을 때도 오랜 시간이 걸리고 처리할 때도 오래 걸린다

	browser는 HTML을 읽다가 <script></script> tag를 만나면 script를 먼저 실행해야하므로 DOM 생성을 멈춘다
	src 속성이 있는 외부 script가 적용되어도 동일하다

	- script에서는 script 아래에 있는 DOM요소에 접근할 수 없다
		DOM 요소에 이벤트 핸들러를 추가하는 등의 사용이 불가능하다
	- page 위쪽에 용량이 큰 script가 있는 경우 script가 page를 막아버린다
		page에 접속하는 사용자들은 script를 다운받고 실행할 때 script 아래 쪽에 있는 page를 볼 수 없다

1) defer(지연 script)
	defer 속성이 있는 script를 백그라운드에서 download 한다
	지연 script를 download 하는 도중에도 HTML 파싱이 멈추지 않고 defer script 실행은 page 구성을 끝날때까지 지연된다
	- 지연 script는 page 생성을 절대 막지 않는다
	- 지연 script는 DOM에 준비된 후에 실행되긴 하지만 DOMConentLoaded 이벤트 발생 전에 실행된다
	※ defer 속성은 외부 script에만 유효하다
	<script defer src="script file 경로"></script>
	src 속성이 없으면 무시된다

2) async(비둘기 script)
	page와 완전히 독립적으로 동작한다

	- 백그라운드에서 download된다
		html page는 async 스크립트 다운이 완료되기를 기다리지 않고 페이지 내 콘텐츠를 처리하고 출력한다
	(async script 실행중에는 HTML 파싱이 멈춘다)
	- DOMContentLoaded 이벤트와 async script는 서로를 기다리지 않는다
		page 구성이 끝난 후에 async script 다운로딩이 끝난 경우 DOMContentLoaded는 async script 실행 전에 발생할 수 있다
